# Master Implementation Plan – Passive Extraction Workflow & State Consistency (Fixes A–G)

## 1) Goals (authoritative)
- Enforce Fresh-Start vs Resume semantics exactly as specified:
  - No heuristics and no category-completion–derived offsets.
  - `system_progression` is the source of truth.
- Run the hybrid pipeline end-to-end (Supplier → Manifest → Filtering → Amazon).
- Manifests must contain the full per-category URL list before any filtering.
- Category iteration honors absolute resume offsets across batches/chunks.
- Invariants remain protective but do not halt legitimate resume sessions; startup counters reconcile to ground truth.

---

## 2) Hybrid Processing Pipeline (Single-Category Cycle)
**Primary file:** `tools/passive_extraction_workflow_latest.py`

### Phase 1 — Category-Level Supplier Processing
**Step 1.1 — URL Discovery & Extraction**
- **Input:** `category_url`
- **Action:** Scrape category page(s); normalize URLs (lowercase host, strip tracking/query params).
- **Persist (P0):** Immediately after extraction, seed the in-memory manifest:
  ```python
  self.category_manifests[category_url] = [
      p.get("url", "") for p in category_products if p.get("url")
  ]
````

* **Anchors:** Category loop \~3838–3846; manifest write \~3874–3876 (**Fix A** guard).

**Step 1.2 — Linking Map Comparison (Skip Complete Products)**

* **Logic:** O(1) set of normalized `supplier_url` from `linking_map.json`.
* **Output:** `skip_entirely[]` (fully complete; no further work), remainder continues.
* **File:** `utils/url_filter.py` (build `linking_map_urls`).

**Step 1.3 — Product Cache Comparison (Skip Supplier Extraction Only)**

* **Logic:** O(1) set of normalized cached product URLs.
* **Output:** `needs_supplier_extraction[]`, `needs_amazon_only[]`.
* **File:** `utils/url_filter.py` (build `cached_urls`).

**Step 1.4 — Supplier Data Extraction**

* **Input:** `needs_supplier_extraction[]`
* **Action:** Extract title, price, EAN, description, images, specs; persist to cache atomically.
* **State:** `current_phase = "supplier"`, update via `update_progression_unified(...)`.

### Phase 2 — Category-Level Amazon Processing

**Step 2.1 — Compile Amazon Processing Queue**

* **Queue:** `amazon_queue = needs_amazon_only + newly_extracted_urls`.

**Step 2.2 — Amazon Product Detail Extraction**

* **Action:** EAN search → fallback to title; extract ASIN, price, reviews, rank, availability; create/merge linking-map entry.
* **Persist:** Atomic `_save_linking_map(...)`; rebuild in-memory hash indexes after save.
* **Anchors:** \~4682–4710 (queue, atomic save & progression updates).

**Step 2.3 — Category Completion**

* **Action:** Mark category done; reset product index; set phase back to `"supplier"`.
* **State:** `current_category_index += 1`; `current_product_index_in_category = 0`; atomic state save.

### Phase 3 — Category Loop Continuation

* Iterate to next category until all processed; chunking permitted but **absolute indices** must be preserved (see Fix C/E).

**D.1 — Zero auxiliary counters on fresh-start**

* **Why:** Prevent inherited offsets from prior runs (or partial states) influencing a true fresh start.
* **Where:** Inside the fresh-start branch that seeds `system_progression` (in `load_or_init_state` or `_calculate_resume_point`).
* **Action:** Clear auxiliary offsets at both the top level and within `supplier_extraction_progress` if present.

  ```python
  def _seed_fresh_start(state: dict, categories: list[str]) -> dict:
      first_url = categories[0] if categories else None

      # Primary seed
      sp = {
          "current_phase": "supplier",
          "current_category_index": 0,
          "current_product_index_in_category": 0,
          "current_category_url": first_url,
          "total_categories": len(categories),
          "total_products_in_current_category": 0,
      }
      state["system_progression"] = sp

      # Zero auxiliary offsets to avoid inherited resume behavior
      aux_keys = ("last_processed_index", "progress_index", "resumption_index")
      for k in aux_keys:
          state[k] = 0

      sep = state.setdefault("supplier_extraction_progress", {})
      for k in aux_keys:
          sep[k] = 0

      # Persist if your flow expects immediate durability here
      # self.state_manager.save_state_atomic()  # if available/appropriate

      return sp
  ```

## 4) State Contract & Precedence (documented behavior)

1. **Operator override** (if provided): `force_fresh_start` or `START_AT_CATEGORY` ⇒ write explicit `system_progression`.
2. **Fresh-Start** (no/minimal state): seed to category index **0**, first URL.
3. **Resume** (valid `system_progression`): resume exactly at `(category_index, product_index_in_category, phase)`.
4. Never derive the start index from category completion counts or the presence of manifests/caches.

---

* Just before manifest write: `manifest_populated: N urls`.
* In chunked mode: `absolute_category_index=...` (and optionally `batch_relative_index=...`).

Operating Workflows

Step-by-step guidance for fresh runs and stale data re-analysis

Fresh Run — Identify Profitable & Sellable Products

Phase 1 — Quick Overview (Dashboard Tab)

  1. Select supplier in sidebar → select Main Workflow lineage
  2. Select the latest CSV in the Financial Report dropdown
  3. Check top metric cards: Profitable (EAN-matched, NetProfit > 0), With Sales (subset with bought_in_past_month > 0), Avg ROI, Avg Profit
  4. Scan charts:
    • ROI Distribution — look for concentration points
    • Profitable Categories — highest total profit bars (count in label)
    • Net Profit vs Selling Price — green dots (ROI ≥ 30%) are best candidates
    • Profit vs Competition — green dots with low total_offer_count = best opportunity

Phase 2 — Detailed Analysis (Analysis Tab)

  1. Set tier filter to T1 → sort by NetProfit desc → review VERIFIED matches
  2. Switch to T2 — review LIKELY matches that need manual check
  3. Apply min_sales ≥ 50 filter to focus on proven demand

Phase 3 — Validate T2/T3 Matches (Operator Tab)

  1. Analysis tab → set Tier filter to T2 (repeat for T3 if needed) → apply NetProfit > 0, Sales ≥ 50 → Export CSV
  2. Copy the downloaded file into OUTPUTS/FBA_ANALYSIS/financial_reports/{supplier}/
  3. Operator tab → select that file from the Financial Report dropdown → click Run AI Analysis
  4. LLM verdict per row: VERIFIED / HIGH LIKELIHOOD = keep  |  NEEDS VERIFICATION or pack mismatch flagged = remove
  5. Review full report in OUTPUTS/CONTROL_PLANE/FINANCIAL_REPORTS/run_{timestamp}/COMBINED_AI_ANALYSIS.md

Phase 4 — Build Final Product List

  1. Analysis tab → filter T1+T2, NetProfit > 0, Sales > 0 → Export CSV
  2. Remove any rows the operator agent flagged as false positives in Phase 3
  3. Remaining list = actionable sourcing list

Phase 5 — Spot-Check Top Items

  1. Open Amazon listing for top 10-20 products by profit
  2. Verify product matches visually, check current price vs report
  3. Confirm Buy Box availability, review count and rating

Stale Data — Identify Categories & Products to Re-Analyze

Phase 1 — Identify Staleness

  1. Check the Financial Report filename date (e.g. 20260122 = Jan 22)
  2. > 14 days = moderately stale  |  > 30 days = significantly stale

Phase 2 — Identify High-Value Re-Analysis Targets

  1. Analysis tab → filter T1+T2, NetProfit > 0, Sales > 0 → sort by Sales desc
  2. High-sales products change fastest — these are top re-check candidates
  3. Export CSV — this is your re-analysis candidate list

Phase 3 — Prioritize Categories

  1. Dashboard → Profitable Categories With Sales chart
  2. Categories with most products (count in label) AND highest total profit = best re-analysis targets
  3. Re-analyzing one category re-scrapes all its products efficiently

Phase 4 — Execute Selective Re-Analysis

  1. Category-level: AI Assistant tab → "Run sandbox analysis for category [URL] on [supplier]" → wait for completion → switch lineage to Latest Sandbox
  2. Product-level: Build product list JSON → place in OUTPUTS/CONTROL_PLANE/inputs/ → enqueue sandbox run via chat

Phase 5 — Compare Old vs New

  1. After sandbox run: switch lineage to Latest Sandbox
  2. Compare NetProfit, Sales, total_offer_count between old and new report
  3. NetProfit stable/improved = confirmed opportunity  |  dropped = deprioritize
Do NOT use an Analysis export as a substitute for running a fresh system scrape — the LLM validates title/EAN/category match quality only, not whether prices or competition have changed. For stale data: always re-run the system for fresh Amazon data first, then validate T2/T3 matches via the Operator tab on the new report.

Prompt Triggers Quick Guide

  1. Main workflow: "run main workflow for [supplier]" / "run all categories for [supplier]"
  2. Category sandbox: "run sandbox analysis for category [URL] on [supplier]"
  3. Category sandbox resume: "resume category sandbox run [run_id]"
  4. Product-list refresh: "refresh product list [products_path] for [supplier]"
  5. Product-list resume: "resume product-list refresh run [run_id] with [products_path]"
  6. If chat was reset/rebooted: include both run_id and products_path explicitly.

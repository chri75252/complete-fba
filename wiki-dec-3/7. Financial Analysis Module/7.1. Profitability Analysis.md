## Introduction
This document provides a comprehensive analysis of the profitability sub-module within the Amazon FBA Agent System, focusing on the calculation of ROI, net profit, and breakeven points. The core functionality is implemented in the `financials()` function within `FBA_Financial_calculator.py`, which combines supplier pricing with Amazon fees, VAT, and prep costs to determine financial viability. The system uses UK VAT-registered business accounting principles, where input VAT is recoverable from HMRC, and output VAT is payable on sales. The analysis covers the complete financial calculation pipeline, from data retrieval through fee extraction to final profitability classification and reporting.

## Core Financial Calculations
The `financials()` function serves as the central engine for profitability analysis, integrating supplier costs with Amazon fees and tax considerations to calculate net profit and ROI. The function begins by retrieving the current Amazon selling price from the scraped product data, prioritizing the `current_price` field when available. If no price is found, the function logs a warning and returns an empty result set to prevent invalid calculations.

Supplier pricing is processed according to the `SUPPLIER_PRICES_INCLUDE_VAT` configuration setting, which determines whether input prices are treated as inclusive or exclusive of VAT. When supplier prices include VAT, the function calculates the ex-VAT cost by dividing the inclusive price by (1 + VAT_RATE). The default VAT rate of 20% is loaded from `system_config.json`, ensuring consistency across calculations.

The total cost structure includes several components: supplier cost (ex-VAT), prep house fee, and shipping cost. The prep house fee is configured in the system settings with a default value of £0.55, while shipping cost defaults to zero. Amazon fees are extracted from Keepa data, with referral fees and FBA fees parsed from the product details section. When Keepa data is unavailable, default values of 15% of the selling price for referral fees and £2.80 for FBA fees are applied.

## ROI and Profit Margin Formulas
The profitability analysis implements two key financial metrics: Return on Investment (ROI) and Profit Margin. The ROI calculation follows the formula `(Net Profit / Total Cost) * 100`, where total cost represents the sum of supplier cost (ex-VAT), prep costs, and shipping costs. This approach ensures that ROI reflects the true investment required to bring a product to market.

The Profit Margin is calculated as `(Net Profit / Revenue) * 100`, providing insight into the profitability relative to sales volume. Both metrics are expressed as percentages to facilitate comparison across products with different price points.

The net profit calculation accounts for all financial flows in the transaction:
- **Net Proceeds**: Selling price (inc VAT) minus referral fee, FBA fee, output VAT, and supplier cost (inc VAT)
- **HMRC**: Output VAT minus input VAT, representing the net VAT payment to tax authorities
- **Net Profit**: Net proceeds minus HMRC, prep house fee, and shipping cost

The breakeven point is calculated as the minimum selling price required to cover all costs: `supplier_price_inc_vat + referral_fee + fba_fee + prep_cost + ship_cost`. This value helps identify products that are currently unprofitable but could become viable with price adjustments.

## Fee Extraction from Keepa Data
The system extracts Amazon fees from Keepa browser extension data through the `extract_keepa_fees()` function, which parses the product details section of the scraped Amazon page. The function searches for key-value pairs containing fee information, specifically targeting "Referral fee" and "FBA fee" fields. To ensure accurate parsing, the function skips percentage-based values and focuses on monetary amounts.

The fee extraction process removes currency symbols (£, $, €) and commas from the values before converting them to floating-point numbers. This normalization allows the system to handle fee data regardless of the specific formatting used by Keepa. The function returns both referral and FBA fees in GBP, maintaining consistency across calculations.

When Keepa data is unavailable or incomplete, the system falls back to default fee structures defined in the configuration. The referral fee defaults to 15% of the selling price, while the FBA fee defaults to £2.80. These defaults ensure that profitability calculations can proceed even when detailed fee data is missing, though results are flagged for review.

## Profitability Thresholds and Classification
The system implements a tiered classification system to categorize products based on their profitability metrics. The thresholds have been adjusted based on market analysis to reflect realistic wholesale margins, as the original thresholds (30-35% ROI) excluded 99.8% of products as unviable.

The adjusted investment tiers are:
- **TIER 1 - IMMEDIATE BUY**: ROI ≥ 25%, Net Profit ≥ £0.50, Sales > 50/month
- **TIER 2 - STRONG CONSIDER**: ROI ≥ 20%, Net Profit ≥ £0.40, Sales > 30/month
- **TIER 3 - INVESTIGATE**: ROI ≥ 15%, Net Profit ≥ £0.30, with additional consideration for strong brands or high sales volume
- **TIER 4 - MONITOR**: ROI ≥ 10%, Net Profit ≥ £0.20, requiring market monitoring for improvement opportunities

These thresholds are implemented in `UK_VAT_Adjusted_Investment_Screening.py`, which applies the classification logic to the financial results generated by the core calculator. The system also considers strong brand indicators (e.g., Disney, LEGO, Apple) as a factor in the investment decision, recognizing that branded products may have more stable demand patterns.

## Results Processing and Export
The `run_calculations()` function orchestrates the financial analysis process, from data retrieval through result generation and export. The function begins by generating supplier-specific paths for input and output files, ensuring that each supplier's data is processed and stored in dedicated directories. This approach prevents data contamination between suppliers and facilitates targeted analysis.

The processing workflow follows these steps:
1. Load supplier product data from the cache
2. For each product, retrieve corresponding Amazon data using EAN, ASIN, or URL matching
3. Extract pricing and fee information from the Amazon data
4. Calculate financial metrics using the `financials()` function
5. Aggregate results into a pandas DataFrame

The results are sorted by ROI in descending order to prioritize the most profitable opportunities. The final dataset is exported to a CSV file in the supplier-specific financial reports directory, with a timestamped filename to ensure uniqueness. The export includes comprehensive product information, financial metrics, and enhanced metrics such as sales volume and seller competition.

## Error Handling and Logging
The profitability analysis system implements comprehensive error handling and logging to ensure data integrity and facilitate troubleshooting. When critical data is missing, such as Amazon price information, the system logs detailed warnings that include the product's EAN, ASIN, and available price fields. These logs help identify patterns in data quality issues and guide improvements to the scraping process.

The system validates file paths and directory structures before processing begins, raising explicit errors if required directories do not exist. This early validation prevents silent failures that could result in lost calculations. During processing, the system tracks statistics on the number of products processed, matches found, and calculations generated, providing visibility into the analysis completeness.

For products that fail financial calculations, the system continues processing other products rather than aborting the entire analysis. This fault-tolerant approach ensures that partial results are available even when some data is problematic. The final output includes a summary of processing statistics, allowing users to assess the quality and completeness of the analysis.

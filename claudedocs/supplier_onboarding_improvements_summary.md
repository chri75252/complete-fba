### 3. AUTHENTICATION_CUSTOMIZATION.md Content Highlights

**Platform-Specific Sections Include**:

#### Shopify (e.g., dkwholesale.com)
- **General Pattern** (to VERIFY): Login URL, form fields, auth indicators
- **What to be most careful about**: Drawer/modal forms, JavaScript-heavy flow, wait times
- **Chrome DevTools MCP investigation commands** (copy-paste ready)
- **Common Pitfalls**: Drawer forms, account dropdowns, JS redirects

#### Magento
- **General Pattern** (to VERIFY): Login URL, form fields with CSRF tokens
- **What to be most careful about**: Form key expiration, theme variations, full page cache
- **Chrome DevTools MCP investigation commands**
- **Common Pitfalls**: Form key expiration, multi-store redirects

#### WooCommerce
- **General Pattern** (to VERIFY): Login URL, form fields, auth indicators
- **What to be most careful about**: Admin bar interference, dual content display, plugin conflicts
- **Chrome DevTools MCP investigation commands**
- **Common Pitfalls**: Dual content, admin bar interference, membership plugins

#### Custom/Proprietary
- **What to be most careful about**: Everything requires manual analysis
- **Chrome DevTools MCP investigation requirements**: Map complete flow, document every selector
- **Common Pitfalls**: CAPTCHA blocks, multi-step flows, unusual storage patterns
- **⚠️ WARNING**: Requires 2-3x more time than platform sites

**Key Principle Emphasized Throughout**:
```
1. NEVER ASSUME - Always verify with Chrome DevTools MCP
2. Document Everything - Keep notes on all selectors found
3. Start Simple - Test basic selectors first, then add complexity
4. Use Screenshots - Capture page state at each step for debugging
5. Test Incrementally - Verify each selector works before moving to next
```
